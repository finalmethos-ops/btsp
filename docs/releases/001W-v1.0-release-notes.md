# BTSP v1.0 Release Notes

## Delivered

- Versioned BPP and Independent purchasing workflows with approval policies and notifications.
- Purchase requests, internal catalog lines, validations, drafts, and attachments.
- Purchase order generation, vendor splitting, artifacts, and transmission lifecycle.
- Vendor acknowledgements, shipment/ASN records, imports, connector scheduling, and operations.
- Receiving, variance, backorder, invoice matching, and reconciliation workflows.
- Operational analytics, exports, vendor scorecards, and scheduled reports.
- Governed user, role, configuration, workflow, notification, health, and audit administration.
- Least-privilege permissions, immutable event evidence, production configuration guards, and operational runbooks.

## Database

BTSP v1.0 spans Alembic revisions `0001_initial_schema` through `0028_audit_reporting`. Upgrade from an earlier deployment with `alembic upgrade head`. Rehearse restore-based rollback against a production-like backup before deployment; schema downgrades can discard data introduced by later epics.

## Compatibility and boundaries

- PostgreSQL and Redis are required production dependencies.
- Durable attachment, purchase-order export, and analytics-report storage must be backed up with the database.
- BTSP records payment approval but does not transmit funds or replace an ERP/general ledger.
- Vendor transports require deployment-specific workers, secret resolution, and partner certification.
- Human release, security, operations, and business approvals remain external governance gates.

## Release status

This worktree is a development candidate, not an immutable release. Publish a v1.0 tag only from a reviewed clean commit after all checks in the 001W.6 validation record and deployment runbook are complete.

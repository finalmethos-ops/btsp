## Live PostgreSQL scenario

The repeatable validator at `backend/scripts/validate_001t_acknowledgements.py`:

1. Created an active vendor, transmitted PO, and inbound vendor endpoint.
2. Recorded an accepted-with-changes acknowledgement envelope.
3. Processed the event into one durable acknowledgement record.
4. Projected the PO to `vendor_acknowledged_changes`.
5. Projected the inbound event to `processed`.
6. Reprocessed the event and received the original acknowledgement without duplication.
7. Confirmed one immutable `vendor.purchase_order_acknowledged` snapshot.
8. Restarted the backend and verified all projected state remained present.

## Decision

**GO for package 001T.3 — Shipment Updates and ASN Domain.**

Acknowledgement processing does not mutate PO lines or financials and does not automatically accept
vendor-proposed changes. Shipment scheduling, ASN processing, receiving, and reconciliation remain
outside this package.

# 001T.6 Security Validation

Validated on 2026-06-28.

## Automated coverage

- Backend suite: 147 tests passed.
- Secret detection covers nested authorization headers, token/key variants, client secrets, and URLs containing user information.
- Worker tests confirm raw lease tokens are not persisted, invalid lease tokens fail, expired leases are recovered, and unsafe legacy endpoint configuration fails closed.
- Ruff formatting and lint passed.
- Frontend formatting, ESLint, TypeScript, and 9 Vitest tests remained clean from 001T.5.

## Database

Migration `0021_connector_security` was upgraded, downgraded to `0020_connector_operations`, and upgraded again on fresh PostgreSQL. Active leases from the pre-hash schema are deliberately invalidated during upgrade.

## Runtime audit

The local runtime was upgraded to migration head and checked with `scripts/validate_001t_security.py`. The audit reports only aggregate counts and fails on credential-like endpoint configuration, malformed or expired active leases, and missing vendor permissions.

## Production qualification boundary

These checks validate BTSP's application and persistence controls. They do not certify a vendor transport or EDI implementation. Each production partner still requires external secret-provider setup, transport supervision, mapping validation, monitoring, and partner acceptance testing.

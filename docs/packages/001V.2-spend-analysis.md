# 001V.2 — Spend Analysis

## Outcome

001V.2 adds filterable, currency-safe landed spend analysis over purchase-order lines.

## Dimensions

- Vendor
- Receiving store
- Purchasing workflow
- Product category, with an explicit `Uncategorized` bucket
- Purchase-order creation month

Every row retains currency, distinct PO count, line count, ordered quantity, and landed line amount. Landed amount is line extension plus allocated freight and tax captured on the immutable PO line.

## Filters

- Inclusive `date_from` and exclusive `date_to`
- Vendor code
- Store number
- Workflow code

## API and UI

- `GET /api/v1/analytics/spend`
- Analytics dashboard: `/analytics`
- Permission: `analytics.read`

Currency is always part of the grouping key. Exchange-rate conversion is intentionally absent until BTSP has an authoritative rate source, valuation date, and accounting policy.

# 001T.6 — Security and Production Validation

## Outcome

001T.6 hardens connector credentials, worker leases, upload boundaries, least-privilege roles, and production operations. It completes the Vendor Integration Platform capability baseline while keeping partner transport workers and secret-provider integrations deployment-specific.

## Security controls

- Worker lease tokens are generated with 256 bits of randomness, returned once, and stored only as SHA-256 digests.
- Migration `0021_connector_security` invalidates active leases created before digest storage.
- Lease result reporting requires the active token and rejects expired or superseded leases.
- Endpoint configuration rejects common secret/credential key variants, nested authorization data, private keys, access tokens, and URLs containing user information.
- Worker claims revalidate persisted endpoint configuration and dead-letter unsafe legacy records.
- Successful execution linkage accepts only a completed import run belonging to the same endpoint.
- Worker claims disclose only transport metadata, non-secret configuration, and the opaque external secret reference.
- Nginx caps request bodies at 11 MiB while the application independently caps connector imports at 10 MiB.
- Nginx emits `X-Content-Type-Options: nosniff` and `Referrer-Policy: same-origin`.

## Production validation

The read-only `scripts/validate_001t_security.py` audit checks endpoint configuration, running lease integrity/expiry, and required system permissions without printing protected values.

Deployment and incident procedures are documented in `docs/runbooks/vendor-connector-operations.md`.

## Explicit production dependencies

BTSP supplies the durable transport-neutral control plane. A production deployment must still provide:

- TLS and an approved identity provider/authentication lifecycle;
- an external secret provider and connector-specific resolver;
- supervised scheduler and worker processes;
- partner-specific REST, SFTP, X12, or EDIFACT mapping/configuration;
- centralized redacted logs, metrics, dead-letter alerts, and on-call ownership;
- vendor certification and end-to-end partner acceptance testing.

EDI remains fail-closed until an explicit partner/version/transaction-set mapping is installed. This is a safety property, not an implied generic EDI parser.

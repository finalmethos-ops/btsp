# Implementation Package 001P.6 — Release Hardening and Deployment Readiness

## Objective

Convert the validated 001P BPP Workflow Platform into a repeatable on-premises release baseline with installation, support, backup, rollback, security, and go-live guidance.

## Scope

This package adds:

- 001P release notes and known-issues register
- Deployment and go-live checklist
- Database-aware rollback plan
- Production deployment runbook
- PostgreSQL backup and restore runbook
- Complete environment-variable and secret guidance
- Docker build, startup, status, and log procedures
- Security and permission review checklist
- Migration order, schema inspection, and rollback limitations
- Consolidated full-release seed idempotency test
- Backend and frontend Docker build-context exclusions

## Migration Review

Current Alembic order:

1. `0001_initial_schema`
2. `0002_identity_rbac`
3. `0003_store_operational_fields`
4. `0004_store_audit_fields`
5. `0005_configuration_entries`
6. `0006_workflow_engine`
7. `0007_workflow_metadata`
8. `0008_notification_framework`

001P adds explicit workflow metadata/state columns and the `notification_templates` and `notification_events` tables. Rollback by database restore is preferred because downgrading these revisions removes persisted release data.

Manual database checks should cover:

- `workflow_definitions` and `workflow_instances`
- `notification_templates` and `notification_events`
- `configuration_entries` scoped to `BPP_PURCHASING`
- `event_snapshots`
- `permissions`, `roles`, and their join tables
- `alembic_version`

## Seed Idempotency

The release test repeats:

- Administrator bootstrap roles and permissions
- Platform configuration defaults
- Workflow registry verification
- BPP purchasing definition and permissions
- BPP approval policy defaults
- BPP notification configuration and templates

It confirms unique users, roles, permissions, definitions, configuration entries, and templates.

## Validation Commands

```bash
docker compose config
docker compose build
docker compose up -d
docker compose ps
docker compose exec backend alembic upgrade head
docker compose exec backend pytest
docker compose exec backend ruff check app tests
docker compose exec frontend npm run build
docker compose exec frontend npm test -- --run
```

Review logs for backend, frontend, PostgreSQL, Redis, and Nginx before acceptance.

## Definition of Done

- All six required release/runbook artifacts exist.
- Environment, security, Docker, migration, seed, rollback, operational, and go-live guidance is complete.
- Full release seed sequence is idempotent.
- Compose configuration/build/startup pass.
- PostgreSQL migration reaches head.
- Container backend/frontend tests and build pass.
- Service logs show no blocking errors.

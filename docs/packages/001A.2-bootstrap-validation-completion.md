# Implementation Package 001A.2 — Bootstrap Validation & Completion

## Objective

Complete and validate the BTSP bootstrap so future packages build on a runnable engineering foundation.

## Scope

This package adds:

- Backend production-safety settings validation
- API readiness endpoint covering PostgreSQL and Redis
- Initial Alembic migration for stores and event snapshots
- Backend pytest and Ruff configuration
- Frontend TypeScript validation configuration
- Next.js environment type file
- GitHub Actions CI for backend, frontend, and Docker Compose
- Local installation runbook

## Validation Targets

```bash
cp .env.example .env
make up
curl http://localhost:8000/api/v1/health
curl http://localhost:8000/api/v1/ready
make migrate
```

Expected health response:

```json
{"status":"ok"}
```

Expected readiness response:

```json
{"status":"ready"}
```

## Notes

This package keeps BTSP aligned to on-premises first deployment while retaining cloud-ready service boundaries.

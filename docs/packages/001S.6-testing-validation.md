# 001S.6 — Purchase Order Testing and Validation

This package closes automated validation for the Purchase Order Engine across numbering, grouping,
splitting, consolidation, immutable artifacts, internal transmission, workflow isolation, API
contracts, and frontend payloads.

## Automated coverage

- Atomic annual numbering and duplicate-source protection.
- Vendor, workflow, currency, request, and optional store grouping boundaries.
- Maximum-line and maximum-total splitting with aggregate financial equality.
- PDF, CSV, and JSON content, checksums, idempotency, and snapshots.
- Internal transmission eligibility, permissions, state transitions, event history, and snapshots.
- BPP and Independent workflow isolation.
- Management-scope and system-administrator route access.
- Generation batch bounds and transmission enum/text constraints.
- Frontend generation and transmission payload contracts.

## Validation commands

```sh
docker compose exec backend ruff check app tests
docker compose exec backend pytest
docker compose exec frontend npm run lint
docker compose exec frontend npm test -- --run
docker compose exec frontend npm run build
docker compose exec backend alembic upgrade head
```

## Release gate

Automated checks are necessary but do not alone close the production gate. Final validation must run
the full migration chain against PostgreSQL, generate and download all artifact formats from the
deployed API, exercise a complete operator-controlled transmission, verify immutable snapshots and
events, and confirm restart persistence for artifact bytes and metadata.

External delivery, vendor acknowledgement, shipment updates, ASN, EDI, and vendor APIs remain out
of scope. Those capabilities begin in Epic 001T.

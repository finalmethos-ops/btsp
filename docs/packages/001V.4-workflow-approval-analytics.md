# 001V.4 — Approval and Workflow Analytics

## Outcome

001V.4 adds cohort-based workflow throughput, approval outcome, and completion-time analytics from durable workflow instances and transition snapshots.

## Metrics

- Instance, active, and completed counts by workflow.
- Current-state distribution.
- Transition, approval, and rejection counts.
- Average, median, and p90 completion time in seconds.
- Approval/rejection counts by transition actor.

## Semantics

- Date filters select instances by workflow start time.
- Completion timing includes only instances whose status is `complete`.
- Active workflow age never enters completion averages.
- Completion time uses terminal instance `updated_at - started_at`.
- Approval actions are transition actions ending in `approve`; rejection is the explicit `reject` action.
- Actor metrics come from immutable `workflow.advanced` snapshots.

## API and UI

- `GET /api/v1/analytics/workflows`
- Optional `date_from`, `date_to`, and `workflow_code` filters.
- Analytics dashboard: `/analytics`
- Permission: `analytics.read`

These are operational process metrics, not employee performance ratings. Production governance should limit actor-level use to workflow capacity, audit, and routing analysis.

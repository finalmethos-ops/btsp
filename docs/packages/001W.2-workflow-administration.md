# 001W.2 — Workflow Administration

## Outcome

001W.2 adds governed visibility and activation controls for versioned workflow definitions.

## Capabilities

- List installed workflow definitions grouped by workflow and version.
- Inspect states, transitions, configuration namespace, and terminal states.
- View active and total instance counts for each pinned version.
- Pause a definition so it accepts no new instances.
- Promote one version atomically, deactivating other active versions of the same workflow.
- Preserve all running instances on the definition version where they started.
- Audit every activation decision with actor, prior state, and superseded versions.

## API and UI

- `GET /api/v1/workflow-admin/definitions`
- `PATCH /api/v1/workflow-admin/definitions/{workflow_code}/versions/{version}/activation`
- Administration workspace: `/admin/workflows`
- Permission: `workflows.manage`

Definitions remain release/configuration managed. This package deliberately does not provide an unrestricted browser state-machine editor; activation is the production governance control, while definition validation and delivery remain in the tested release path.

No migration is required because workflow definitions, instances, and audit snapshots use existing platform tables.

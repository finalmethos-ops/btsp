# 001S.1 — Purchase Order Foundation

## Delivered

- Purchase Order header, source-link, line, and annual sequence persistence.
- Atomic configurable numbering.
- Eligibility enforcement at workflow state `po_created`.
- Workflow-specific generation permission enforcement.
- Duplicate source-request protection.
- Same-workflow/vendor/currency consolidation with a configuration switch.
- Transactional copies of all financial line values.
- Immutable `purchase_order.created` snapshot.
- Protected seed, generation, list, and detail APIs.

## API

- `POST /api/v1/purchase-orders/seed-defaults` — requires `configuration.manage`.
- `POST /api/v1/purchase-orders/generate` — accepts one to 100 Purchase Request IDs.
- `GET /api/v1/purchase-orders`.
- `GET /api/v1/purchase-orders/{id}`.

BPP generation requires `workflow.bpp.po_generate`. Independent generation currently uses the
existing `workflow.ind.review` permission aligned with its `generate_po` transition. List and detail
visibility remain restricted to the user's BPP or Independent administrative scope.

## Configuration

| Key | Default |
| --- | --- |
| `purchase_order / default / numbering` | `{"prefix":"PO","padding":6}` |
| `purchase_order / default / consolidation.enabled` | `{"enabled":true}` |
| `purchase_order / default / consolidation.by_store` | `{"enabled":false}` |
| `purchase_order / default / split.max_lines` | `{"count":0}` |
| `purchase_order / default / split.max_total` | `{"amount":0}` |

## Validation

docker compose exec backend pytest

Automated tests cover numbering, financial copies, source links, snapshots, permission rejection,
duplicate generation rejection, and consolidation.

# 001S.5 — Purchase Order Frontend

The authenticated `/purchase-orders` workspace provides workflow-scoped PO operations without
duplicating backend business rules.

## Capabilities

- BPP/Independent scoped PO list and detail.
- PO number, status, vendor, source request, store, line, and total visibility.
- PDF, CSV, and JSON generation and authenticated download.
- Source workflow advancement from `po_created` to `vendor_submission` after PO generation.
- Internal transmission preparation with artifact/channel/destination/audit note.
- Release, delivered, failed, retry, and cancel controls based on current transmission status.
- Transmission audit-event counts and surfaced API validation errors.
- Purchase Request `po_created` action linking directly into PO generation.
- Dashboard navigation for authorized PO administrators.

The interface records internal handoff only. It does not contact a vendor, send email, invoke EDI,
or assert external delivery. Backend workflow, permission, artifact, and state-machine validation
remain authoritative.

## Validation

```sh
docker compose exec frontend npm run lint
docker compose exec frontend npm run build
docker compose exec frontend npm test -- --run
```

Frontend contract tests cover source generation payloads and transmission action/reason payloads.

# 001V.5 — Inventory Position, Exports, and Scheduled Reports

## Outcome

001V.5 adds a cumulative receiving position, immediate CSV exports, and durable scheduled analytics report runs with verifiable artifacts.

## Inventory position

- Groups posted receipt quantities by store and product.
- Reports cumulative accepted and rejected quantities.
- Reports outstanding quantities from open and partially fulfilled backorders.
- Supports store, product, and vendor filters.

This is a receiving and backorder position, not true on-hand inventory. Stock adjustments, transfers, reservations, and depletion require a future inventory ledger before an on-hand claim is valid.

## Reports and exports

- Immediate CSV exports are available for inventory position, spend, vendor scorecards, and workflow analytics.
- Administrators can create interval-based report schedules with stored filters.
- Due executions create immutable run records and CSV artifacts.
- Completed artifacts record content type, byte size, and SHA-256 digest.
- A unique schedule and scheduled-time constraint prevents duplicate interval execution.

## API and UI

- `GET /api/v1/analytics/inventory-position`
- `GET /api/v1/analytics/exports/{report_type}`
- `POST/GET /api/v1/analytics/report-schedules`
- `POST /api/v1/analytics/report-runs/run-due`
- `GET /api/v1/analytics/report-runs`
- `GET /api/v1/analytics/report-runs/{run_id}/content`
- Analytics dashboard: `/analytics`
- Read permission: `analytics.read`
- Schedule/run permission: `analytics.reports.manage`

`ANALYTICS_MANAGER` receives both permissions; `ANALYTICS_VIEWER` remains read-only.

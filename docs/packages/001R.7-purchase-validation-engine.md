# 001R.7 — Purchase Validation Engine

`GET /api/v1/purchase-requests/{id}/validation` returns `ready`, structured `errors`, and structured
`warnings`. Submission invokes the same evaluator and fails when `ready` is false.

Current validation covers:

- Active, orderable Official Store Database reference.
- Active internal catalog vendor.
- Active and available products.
- Catalog minimum quantity.
- Required line items and calculated order minimum.
- Maximum line quantity.
- Store, vendor, product, category, and brand restrictions.
- Allowed region and buying-group restrictions.
- Configured required attachment categories.
- Draft status and expiration.
- Active registered workflow definition.
- Workflow enabled configuration.
- Safe rejection of malformed purchasing configuration.

Required attachment categories are configured with
`purchasing / {workflow_code} / rules.required_attachment_categories` and a value such as:

```json
{"categories": ["quote"]}
```

Changing configuration changes readiness and submission behavior without an application deployment.

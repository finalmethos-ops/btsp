# 001R.10 — Testing and Validation

## Scope

This package closes the Core Purchasing Domain Model release gate across:

- Purchase Request CRUD and workflow lifecycle projection.
- Internal Excel vendor/product catalog ingestion and lookup.
- Line validation, price snapshots, freight, tax, and totals.
- Configuration-driven restrictions and readiness.
- Draft revision control, cloning, recovery, and expiration.
- Managed attachments and immutable snapshots.
- BPP and Independent workflow integration.
- Authentication, ownership, region, and workflow permission boundaries.
- Shared purchasing frontend and production container build.

## Validation commands

```sh
docker compose -f docker-compose.yml -f docker-compose.production.yml config --quiet
docker compose -f docker-compose.yml -f docker-compose.production.yml build
docker compose -f docker-compose.yml -f docker-compose.production.yml up -d
docker compose -f docker-compose.yml -f docker-compose.production.yml exec backend alembic upgrade head
docker compose -f docker-compose.yml -f docker-compose.production.yml exec backend pytest
docker compose -f docker-compose.yml -f docker-compose.production.yml exec backend ruff check app tests
docker compose -f docker-compose.yml -f docker-compose.production.yml exec frontend npm run build
docker compose -f docker-compose.yml -f docker-compose.production.yml exec frontend npm test -- --run
```

## Acceptance criteria

- All migrations and quality gates pass in Linux containers.
- Excel catalog import is deterministic and preserves transactional line pricing.
- A real Purchase Request advances from draft through completed.
- Purchase Request status follows its workflow lifecycle.
- Configuration changes alter readiness without code changes.
- Permission, ownership, region, file type, and invalid-transition controls remain enforced.
- Attachments and domain/workflow events produce immutable snapshots.
- Both purchasing frontend routes build and load through Nginx.
- BPP and Independent workflows remain distinct while sharing the purchasing aggregate.

Detailed evidence and the go/no-go decision are recorded in
`docs/validation/001R-production-validation.md`.

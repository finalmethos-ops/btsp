# Implementation Package 001P.2 — BPP Purchasing Workflow Definition

## Objective

Install the first production business workflow on the reusable BTSP workflow engine.

## Scope

This package adds:

- Registered `BPP_PURCHASING` workflow capability
- Version 1 BPP purchasing state and transition definition
- Explicit workflow state catalog and business metadata persistence
- Fifteen BPP workflow permissions
- Six BPP purchasing configuration defaults
- Deterministic, repeatable seed service
- Protected seed endpoint
- Additive workflow-definition metadata migration
- Contract, idempotency, permission, and engine guard tests
- BPP purchasing workflow documentation

## API Surface

- `POST /api/v1/workflow-engine/seeds/bpp-purchasing`

The endpoint requires `configuration.manage` and uses the authenticated user as the configuration-change actor.

## Seed Behavior

Each invocation:

1. Verifies the code-owned `BPP_PURCHASING` registry entry.
2. Creates or updates all BPP workflow permissions.
3. Assigns those permissions to existing `SYSTEM_ADMIN` and `BPP_ADMIN` roles.
4. Creates or updates definition version 1.
5. Creates or updates all six scoped configuration defaults.

Repeated calls do not create duplicate permission, definition, or configuration records.

## Architecture Rules Supported

- BPP purchasing remains separate from Independent ordering.
- Workflow definitions are versioned.
- Every state-changing action declares a permission.
- Variable thresholds and behavior are configuration-backed.
- Workflow activity and configuration changes retain snapshot behavior.

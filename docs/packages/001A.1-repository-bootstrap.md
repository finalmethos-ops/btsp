# Implementation Package 001A.1 — Repository Bootstrap

## Objective

Establish the permanent BTSP engineering workspace and production-ready application foundation.

## Scope

This package introduces:

- Repository documentation and baseline conventions
- Docker Compose local stack
- FastAPI backend scaffold
- SQLAlchemy database session
- Alembic migration framework
- PostgreSQL service
- Redis service
- Nginx reverse proxy
- Next.js frontend scaffold
- Environment template
- Developer Makefile
- Basic backend health test

## Architecture Rules Captured

BTSP development must preserve the following rules:

1. Official Store Database is the single source of truth.
2. Single login with role-based routing.
3. Event Snapshots are immutable.
4. Configuration over code.
5. Region-locked multi-store ordering.
6. BPP and Independent workflows remain separate.
7. Every package must be production-ready and commit-ready.

## Validation

Expected local validation after cloning:

```bash
cp .env.example .env
make up
```

Health endpoint:

```bash
curl http://localhost:8000/api/v1/health
```

Expected response:

```json
{"status":"ok"}
```

## Notes

Some small framework boilerplate files may need to be finalized in the next package if connector write filtering blocks them. The core repository, backend, infrastructure, and primary frontend scaffold are in place.

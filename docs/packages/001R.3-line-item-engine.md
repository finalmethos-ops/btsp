# 001R.3 — Line Item Engine

The purchase request aggregate supports adding, replacing, and deleting draft line items. Each
operation validates the internal catalog product, owning vendor, availability, and minimum order
quantity, then recalculates subtotal, freight, tax, and total.

Unit price and product name are copied into the line as transactional evidence. Catalog price
changes affect later edits and clones, but never silently rewrite existing line prices.

Draft mutations increment a revision number. Clients may send `expected_revision` when patching a
request; stale edits are rejected so auto-save cannot overwrite a newer update.

# 001R.5 — Draft Management

- `PATCH /api/v1/purchase-requests/{id}` supports manual save and auto-save.
- `expected_revision` provides optimistic concurrency protection.
- `GET /api/v1/purchase-requests?request_status=draft` recovers a user's current drafts.
- `POST /api/v1/purchase-requests/{id}/clone` creates a new draft using current catalog pricing.
- `POST /api/v1/purchase-requests/drafts/expire` expires overdue drafts and requires
  `configuration.manage`.

Draft lifetime is configured by `draft.expiration_days`, defaulting to 30 days per purchasing
workflow. Expiration writes an immutable `purchase_request.expired` snapshot; cloning writes
`purchase_request.cloned`. Submitted or expired requests cannot be edited as drafts.

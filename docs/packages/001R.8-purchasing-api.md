# 001R.8 — Purchasing API

The purchasing API exposes the full initial aggregate lifecycle:

- Purchase-request create, owned listing, read, patch, delete, clone, validation, and submission.
- Line-item add, replace, and delete.
- Internal catalog vendor and product lookup.
- Attachment upload, list, authenticated download, and draft deletion.
- Ownership-protected workflow-instance status for the request timeline.
- Administrative purchasing-default seed and stale-draft expiration operations.

Non-administrators see only requests they created. Workflow-specific submit permissions are enforced
at submission, while catalog management and draft-expiration operations retain administrative
protection. Readiness preview and submission use the same domain validator.

# 001R.9 — Purchase Request Frontend

Both BPP and Independent workflow routes use the shared purchase-request workspace while retaining
their separate workflow codes and submit permissions.

The workspace provides:

- New request creation using internal catalog vendors.
- Existing request and draft recovery.
- Catalog product search and vendor-scoped selection.
- Quantity, freight, tax, and calculated line/aggregate totals.
- Line deletion during draft.
- Readiness errors sourced directly from the backend validation engine.
- Submit and clone operations.
- Attachment upload, authenticated download, and draft deletion.
- Current workflow state and update time.

The browser does not duplicate purchasing eligibility rules. It uses backend results as the
authoritative contract, preventing client and server behavior from drifting as configuration changes.

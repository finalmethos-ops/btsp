# 001W.1 — Role Management

## Outcome

001W.1 adds governed custom-role management and replaces the user editor's fixed role list with the live role catalog.

## Capabilities

- List system and custom roles with permission and assignment counts.
- List the registered permission catalog and descriptions.
- Create and update custom roles with optional workflow scope.
- Assign allow-listed permissions to custom roles.
- Delete unassigned custom roles.
- View protected system roles without modifying them.
- Record role creation, update, and deletion in event snapshots.
- Reject unknown roles during user creation or update instead of silently dropping them.

## API and UI

- `GET/POST /api/v1/roles`
- `GET /api/v1/roles/permissions`
- `PATCH/DELETE /api/v1/roles/{code}`
- Administration workspace: `/admin/roles`
- Permission: `roles.manage`

The `SYSTEM_ADMIN` role receives `roles.manage` through the core permission seed. Custom role codes are stable uppercase identifiers and cannot be renamed after creation.

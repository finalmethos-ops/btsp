# 001T.1 — Vendor Integration Foundation

## Delivered

- Vendor endpoint registry tied to active internal catalog vendors.
- REST API, SFTP, EDI, and manual-import transport classifications.
- Inbound, outbound, and bidirectional endpoint direction controls.
- Opaque connection references with recursive embedded-secret rejection.
- Durable acknowledgement, shipment-update, and ASN event envelopes.
- Canonical payload hashing and endpoint-scoped external ID idempotency.
- Conflict rejection when an external ID is reused with different content.
- Immutable endpoint-creation and event-receipt snapshots.
- Management, read, and ingestion permissions.

## API

- `POST /api/v1/vendor-integrations/endpoints`
- `GET /api/v1/vendor-integrations/endpoints`
- `GET /api/v1/vendor-integrations/endpoints/{id}`
- `POST /api/v1/vendor-integrations/events`
- `GET /api/v1/vendor-integrations/events`

The event API is authenticated BTSP infrastructure. It is not yet a public vendor webhook and does
not claim connector authentication, acknowledgement projection, shipment projection, or ASN
processing.

## Validation

```sh
docker compose exec backend alembic upgrade head
docker compose exec backend ruff check app tests scripts alembic
docker compose exec backend pytest
```

Automated coverage verifies active-vendor enforcement, transport and direction constraints,
recursive embedded-secret rejection, canonical JSON hashing, identical replay, conflicting replay,
event filtering, and immutable snapshots.

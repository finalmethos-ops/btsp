# 001R.4 — Purchasing Business Rules

Rules are configuration entries with scope type `purchasing` and scope key equal to
`BPP_PURCHASING` or `IND_PURCHASING`.

| Key | Value field | Default |
| --- | --- | --- |
| `rules.enabled` | `enabled` | `true` |
| `rules.minimum_order_amount` | `amount` | `0` |
| `rules.maximum_line_quantity` | `quantity` | `10000` |
| `rules.blocked_stores` | `store_numbers` | `[]` |
| `rules.blocked_vendors` | `vendor_codes` | `[]` |
| `rules.blocked_products` | `product_codes` | `[]` |
| `rules.blocked_categories` | `categories` | `[]` |
| `rules.blocked_brands` | `brands` | `[]` |
| `rules.allowed_regions` | `region_codes` | `[]` |
| `rules.allowed_buying_groups` | `buying_group_codes` | `[]` |

Empty allow-lists mean unrestricted. Invalid configuration fails readiness rather than silently
allowing an order. Defaults are installed idempotently with
`POST /api/v1/purchase-requests/seed-defaults`, protected by `configuration.manage`.

`GET /api/v1/purchase-requests/{id}/validation` returns structured errors and warnings. Submission
uses the same evaluator, so preview and enforcement cannot drift.

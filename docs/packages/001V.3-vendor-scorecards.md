# 001V.3 — Vendor Scorecards

## Outcome

001V.3 provides transparent vendor performance components without inventing an opaque weighted score.

## Metrics

- Purchase-order volume and ordered spend by currency.
- Acknowledgement coverage and accepted/rejected outcome counts.
- On-time delivery rate for delivered shipments having both estimated and actual delivery timestamps.
- Receiving acceptance rate from accepted versus rejected physical quantity.
- Invoice line match rate.
- Approved and rejected reconciliation counts.

## Rate semantics

- Rates are percentages rounded to two decimals.
- Zero numerator with a non-zero eligible denominator is `0.00%`.
- Zero denominator is `null` and displayed as `Not measured`; unknown evidence is not treated as poor performance.
- Date filters select purchase orders first, then evaluate associated vendor activity for that cohort.

## API and UI

- `GET /api/v1/analytics/vendor-scorecards`
- Optional `date_from`, `date_to`, and `minimum_orders` filters.
- Analytics dashboard: `/analytics`
- Permission: `analytics.read`

Any future composite score must define policy-owned weights, minimum sample sizes, missing-data treatment, and versioned calculation rules.

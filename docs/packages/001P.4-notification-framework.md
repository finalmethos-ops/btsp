# Implementation Package 001P.4 — Notification Framework

## Objective

Add configurable workflow notifications with persistent templates, event history, recipient resolution, and channel delivery contracts.

## Scope

This package adds:

- Notification template and event database models
- Additive migration `0008_notification_framework`
- In-app, email, and webhook channel abstractions
- Six recipient strategies
- Safe context-based template rendering
- Template create, update, and list services
- Notification emission and event-history services
- Sent and failed delivery update services
- Immutable emitted and failed snapshots
- Four notification permissions
- Five BPP notification configuration defaults
- Eight idempotent BPP in-app templates
- Six protected API endpoints
- Database-backed framework and seed tests
- Notification framework workflow documentation

## API Surface

- `GET /api/v1/notifications/templates`
- `POST /api/v1/notifications/templates`
- `PATCH /api/v1/notifications/templates/{template_code}`
- `GET /api/v1/notifications/events`
- `POST /api/v1/notifications/emit`
- `POST /api/v1/notifications/seeds/bpp-purchasing`

## Architecture Rules Supported

- Recipients and message content are template/configuration-backed.
- API-submitted notification actors are replaced with authenticated identity.
- Notification event history is persistent and queryable.
- Emission outcomes write immutable snapshots.
- Email and webhook integrations remain replaceable delivery adapters.
- BPP notification defaults remain workflow-scoped.

## Validation Commands

```bash
cd backend
alembic upgrade head
pytest
ruff check app tests
```

## Definition of Done

- Templates can be created and updated.
- All six recipient strategies have service contracts.
- Context values render into subject and body templates.
- In-app and email events queue successfully.
- Disabled webhook delivery is skipped safely.
- Inactive templates are skipped.
- Emitted and failed outcomes create snapshots.
- BPP configuration and templates seed without duplicates.
- Migration, tests, and Ruff pass cleanly.

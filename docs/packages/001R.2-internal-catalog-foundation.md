# 001R.2 — Internal Excel Catalog Foundation

## Decision

BTSP uses an internal vendor/product catalog loaded from Excel. No external vendor client or live
vendor integration is part of this package.

## API

- `POST /api/v1/catalog/imports` — `.xlsx`, maximum 10 MB, requires `configuration.manage`.
- `GET /api/v1/catalog/vendors` — authenticated catalog lookup.
- `GET /api/v1/catalog/products` — authenticated search with vendor, active, and result-limit filters.

The import response records filename, actor, completion state, and row counts. A successful import
also writes an immutable `catalog.imported` snapshot.

## Operational procedure

1. Validate vendor and product codes in the canonical workbook.
2. Back up PostgreSQL before a production catalog replacement.
3. Upload the workbook through the protected import endpoint.
4. Verify row counts and query representative vendor/product codes.
5. Review the `catalog.imported` snapshot.
6. Correct the workbook and re-import if reference data must change.

The importer validates the complete workbook before catalog writes and performs deterministic
upserts. Repeated imports are safe and do not duplicate stable codes.

## Acceptance evidence

Automated tests cover deterministic re-import, current-price update, invalid vendor rejection,
minimum quantity enforcement, transactional price preservation, totals, submission, and snapshots.

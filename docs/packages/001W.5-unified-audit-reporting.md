# 001W.5 — Unified Audit Reporting and Export

## Outcome

001W.5 turns the immutable event-snapshot ledger into a searchable audit investigation and reporting surface.

## Capabilities

- Filter by exact event type, entity type, entity identifier, actor, and half-open date window.
- Page through deterministic reverse-chronological results with total counts.
- Summarize the top event types, entity types, and actors for the same active filters.
- Inspect structured evidence payloads without mutating ledger records.
- Export up to 10,000 filtered events as deterministic CSV.
- Escape spreadsheet-formula text in every exported text column.

## API and UI

- `GET /api/v1/audit/events`
- `GET /api/v1/audit/summary`
- `GET /api/v1/audit/export`
- Administration workspace: `/admin/audit`
- Read permission: `snapshots.read`
- Export permission: `audit.export`

List limits are constrained to 1–500, offsets are bounded, invalid date windows return HTTP 422, and exports are independently permissioned.

## Persistence

Migration `0028_audit_reporting` adds indexes for reverse chronological access and actor/time investigations. Event snapshots remain append-only; this package adds no update or delete path.
